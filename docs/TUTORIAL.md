# K-LabDoc 사용 튜토리얼

## 1. 실행 방법

### (A) Railway 서버(배포판)
배포된 URL 접속 → 로그인 화면.

### (B) 로컬 서버
```bash
npm install
npm start        # http://localhost:3000
```

### (C) 서버 없이 (오프라인 데모)
`public/index.html` 파일을 브라우저에서 직접 엽니다. 로컬 PIN(`0000`)으로 로그인됩니다.

## 2. 로그인
1. 상단 탭에서 **전문의 / 임상병리사** 그룹 선택.
2. 사용자 카드 클릭(예: 윤_진단, 최_파트).
3. PIN 입력(초기 `0000`) → **로그인**.
   - 서버 배포판: Railway 인증 서버가 ID+PIN을 검증하고 JWT를 발급합니다.
   - 로컬 파일: 브라우저 내에서 검증(오프라인).

## 3. 핵심 워크플로우

### 지침서 관리
- **지침서 관리 → 지침서** 목록에서 `열기`(내용·현재본), `이력`(작성·개정 이력, 전자서명, 개정 내용 확인), `개정`(새 버전 작성).
- 개정 저장 시 상태가 `검토` 대기로 전환 → 전문의/파트장 **검토 전자서명** → 전문의 **승인 전자서명** → 발효.
- **가져오기(HWP/Word)** 로 기존 지침서를 불러와 열람·검색·인쇄.

### 교육 과정
- **교육 과정 → 맞춤 교육 과정** 에서 역할별 과정 학습 후 **퀴즈(만점 시 이수)**.
- 이수 시 **이수증(PDF)** 발급. 관리자는 **팀 이수 현황** 확인.

### 인증·품질
- **인증·품질 → 인증 준비도** 에서 진단검사의학재단 점검표 문항별 **예/부분/아니오/해당없음** 판정 → 분야별·종합 준비도 점수.
- **실사 대응** 에서 문항 선택 시 근거자료 즉시 취합·인쇄.

## 4. 계정 관리
- **PIN 변경**: 로그인 후 마이페이지(또는 API `POST /api/auth/change-pin`).
- **사용자 추가**: 전문의 권한으로 `POST /api/users` (docs/API.md 참조).

## 5. 데이터 백업
- **설정 → 데이터·설정 → JSON 내보내기/가져오기** 로 검사실 데이터를 백업·이관.
- 검사실 데이터는 브라우저 localStorage에 저장되므로, 브라우저/기기별로 분리됩니다.

## 6. 문제 해결
- 로그인 시 "서버 연결 오류": 서버가 실행 중인지, `/api/health`가 응답하는지 확인.
- "ID 또는 PIN이 올바르지 않습니다": 시드 PIN은 `0000`. 변경했다면 새 PIN 사용.
- 재배포 후 계정 초기화: PostgreSQL 미연결(메모리 모드)일 때 발생 → `DATABASE_URL` 연결.
